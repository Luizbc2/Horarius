# Checklist Basico de Usabilidade

Documento simples para registrar a avaliacao manual de usabilidade do backend integrado ao frontend do Horarius.

## Objetivo

Cobrir o tipo de teste de usabilidade exigido na rubrica com uma verificacao objetiva dos fluxos principais.

## Fluxos avaliados

- cadastro de usuario
- login
- atualizacao de perfil
- cadastro de cliente
- cadastro de servico
- cadastro de profissional
- configuracao de horarios do profissional
- criacao de agendamento

## Checklist

- o usuario entende rapidamente o que deve preencher em cada formulario
- mensagens de erro aparecem em portugues e de forma coerente
- campos obrigatorios ficam claros
- o fluxo de login redireciona corretamente para a area protegida
- o perfil deixa claro que o e-mail nao pode ser alterado
- as listagens possuem busca e paginacao visiveis
- o fluxo de agendamento permite encontrar cliente, servico e profissional sem confusao
- a tela de horarios do profissional deixa claro quais dias estao ativos

## Resultado esperado

Os fluxos principais devem ser executados sem ambiguidade, com mensagens compreensiveis e sem exigir conhecimento tecnico do usuario final.

## Observacao

Este checklist complementa os testes automatizados de modulo, integracao, sistema, seguranca e performance leve.
